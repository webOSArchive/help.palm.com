Ajout d'un profil VPN à un client VPNC

Avant de commencer

Communiquez avec votre fournisseur de courriel ou votre administrateur système pour obtenir les renseignements suivants :

  • Nom de profil
  • Serveur VPN
  • Nom d'utilisateur et mot de passe
  • ID de groupe
  • Code secret du groupe
  • Domaine
  • Paramètre DPD (Dead Peer Dectection)
  • Mode de cryptage : Sécurisé, Faible ou Aucun
  • Mode Nat traversal : Cisco-UDP, NAT-T (toujours), NAT-T (détection auto) ou Désactivé

BYB Close

  1. Ouvrez ! VPN  > Ajouter un profil.
  2. Si VPNC n'est pas déjà sélectionné dans le champ Type de connexion, appuyez sur le champ Type de connexion > VPNC.
  3. Appuyez sur Entrez le nom du serveur > saisissez le nom du serveur.
  4. Appuyez sur Suivant > saisissez les données reçues de votre administrateur de système > Se connecter.
  5. Au besoin, saisissez votre jeton de sécurité ou autre mot de passe > Terminé
  6. Appuyez sur Continuer.

REMARQUE :

Dead Peer Detection : lorsque cette option est sélectionnée, si aucun trafic VPN n'est reçu pendant cinq minutes, le client vérifie si le serveur VPN est toujours disponible.

Méthode de cryptage : appuyez sur une méthode de cryptage; l'option Sécurisé force le client à utiliser des algorithmes AES ou 3DES présentant une sécurité renforcée. L'option Faible permet d'utiliser l'algorithme 1DES plus faible. Le cryptage peut être renforcé en fonction des décisions prises par le serveur VPN. L'option Aucun n'active aucun cryptage. Le cryptage est désactivé; il peut toutefois être utilisé en fonction du serveur VPN. L'option Aucun permet tout simplement au profil VPN de n'utiliser aucun cryptage.
